## 1. Headless 组件模式

### 核心思想
将组件的**逻辑**与**UI 表现**完全分离，组件只负责状态管理和交互逻辑，通过作用域插槽（scoped slots）或 render props 将状态和操作暴露给消费者，由消费者决定如何渲染。

### 适用场景
- 需要高度定制化 UI 的通用组件（如下拉菜单、弹窗、标签页等）
- 设计系统需要支持多套视觉风格
- 组件库需要满足不同业务方的定制需求

### 代码示例

```vue
<!-- HeadlessSelect.vue -->
<script setup lang="ts">
import { ref, computed } from 'vue'

const props = defineProps<{
  options: { label: string; value: any }[]
  modelValue?: any
}>()

const emit = defineEmits<{
  'update:modelValue': [value: any]
}>()

const isOpen = ref(false)
const selectedIndex = computed(() =>
  props.options.findIndex(o => o.value === props.modelValue)
)

const open = () => (isOpen.value = true)
const close = () => (isOpen.value = false)
const select = (value: any) => {
  emit('update:modelValue', value)
  close()
}

// 暴露所有状态和方法
defineSlots<{
  default(props: {
    isOpen: boolean
    options: typeof props.options
    selectedValue: any
    selectedIndex: number
    open: () => void
    close: () => void
    select: (value: any) => void
  }): any
}>()
</script>

<template>
  <slot
    :is-open="isOpen"
    :options="options"
    :selected-value="modelValue"
    :selected-index="selectedIndex"
    :open="open"
    :close="close"
    :select="select"
  />
</template>
```

**消费者使用：**

```vue
<HeadlessSelect v-model="selected" :options="options">
  <template #default="{ isOpen, options, selectedValue, open, select }">
    <div class="my-custom-select">
      <button @click="open">{{ selectedValue ?? '请选择' }}</button>
      <ul v-if="isOpen">
        <li v-for="opt in options" :key="opt.value" @click="select(opt.value)">
          {{ opt.label }}
        </li>
      </ul>
    </div>
  </template>
</HeadlessSelect>
```

### 注意事项
- 过度使用会导致模板嵌套过深，影响可读性
- 对于简单场景，Headless 模式可能显得冗余
- 需要良好的 TypeScript 类型定义来提供良好的开发体验
- 考虑提供预设的"有头"组件版本，方便快速使用

## 2. Composable 的依赖注入模式

### 核心思想
通过 `provide/inject` 将 Composable 的状态和方法注入到组件树中，让深层子组件可以直接访问，避免逐层传递 props（prop drilling），同时保持响应式。

### 适用场景
- 表单系统中的字段状态共享
- 多级嵌套组件间的状态同步
- 大型组件树的跨层级通信

### 代码示例

```vue
<!-- useFormContext.ts -->
import { createInjectionKey, ref, computed, type Ref } from 'vue'

interface FormContext {
  values: Ref<Record<string, any>>
  errors: Ref<Record<string, string>>
  setField: (name: string, value: any) => void
  validate: () => boolean
  reset: () => void
}

// 使用类型安全的注入 key
const FormKey = createInjectionKey<FormContext>('form')

// Provider Composable
export function useFormProvider(initialValues: Record<string, any> = {}) {
  const values = ref({ ...initialValues })
  const errors = ref<Record<string, string>>({})

  const setField = (name: string, value: any) => {
    values.value[name] = value
  }

  const validate = () => {
    // 验证逻辑
    return Object.keys(errors.value).length === 0
  }

  const reset = () => {
    values.value = { ...initialValues }
    errors.value = {}
  }

  const context: FormContext = {
    values,
    errors,
    setField,
    validate,
    reset
  }

  provide(FormKey, context)

  return context
}

// Consumer Composable
export function useFormContext() {
  const context = inject(FormKey)
  if (!context) {
    throw new Error('useFormContext must be used within a FormProvider')
  }
  return context
}
```

**在组件中使用：**

```vue
<!-- Form.vue -->
<script setup lang="ts">
import { useFormProvider } from './useFormContext'

const { values, validate, reset } = useFormProvider({
  username: '',
  email: ''
})
</script>

<template>
  <form @submit.prevent="validate">
    <slot />
  </form>
</template>
```

```vue
<!-- FormField.vue（深层子组件） -->
<script setup lang="ts">
import { useFormContext } from './useFormContext'

const props = defineProps<{ name: string }>()
const { values, errors, setField } = useFormContext()
</script>

<template>
  <div class="form-field">
    <input
      :value="values[props.name]"
      @input="setField(props.name, $event.target.value)"
    />
    <span v-if="errors[props.name]" class="error">
      {{ errors[props.name] }}
    </span>
  </div>
</template>
```

### 注意事项
- 使用 `createInjectionKey` 或 Symbol 保证类型安全和避免命名冲突
- 在 inject 时进行空值检查，提供友好的错误提示
- 注意 provide 的响应式数据在组件树中的生命周期
- 考虑使用 `InjectionKey<T>` 配合 TypeScript 提供更好的类型推导

## 3. 受控与非受控组件的统一设计

### 核心思想
组件同时支持受控（由父组件管理状态）和非受控（内部管理状态）两种模式，通过检测是否传入 `v-model` 或 `value` prop 来决定使用哪种模式，提供灵活的使用方式。

### 适用场景
- 表单输入组件（Input、Select、Checkbox 等）
- 需要同时支持简单场景（非受控）和复杂场景（受控）的通用组件
- 组件库的基础组件设计

### 代码示例

```vue
<!-- SmartInput.vue -->
<script setup lang="ts">
import { ref, computed, watch } from 'vue'

const props = withDefaults(
  defineProps<{
    modelValue?: string
    defaultValue?: string
  }>(),
  {
    defaultValue: ''
  }
)

const emit = defineEmits<{
  'update:modelValue': [value: string]
  change: [value: string]
}>()

// 判断是否为受控模式
const isControlled = computed(() => props.modelValue !== undefined)

// 内部状态（非受控模式使用）
const internalValue = ref(props.defaultValue)

// 统一的当前值
const currentValue = computed(() =>
  isControlled.value ? props.modelValue! : internalValue.value
)

// 统一的值更新方法
const updateValue = (newValue: string) => {
  if (!isControlled.value) {
    internalValue.value = newValue
  }
  emit('update:modelValue', newValue)
  emit('change', newValue)
}

// 受控模式下，同步外部值
watch(
  () => props.modelValue,
  (newVal) => {
    if (isControlled.value && newVal !== internalValue.value) {
      internalValue.value = newVal
    }
  }
)
</script>

<template>
  <input
    :value="currentValue"
    @input="updateValue(($event.target as HTMLInputElement).value)"
  />
</template>
```

**使用方式：**

```vue
<!-- 非受控模式 -->
<SmartInput default-value="hello" @change="handleChange" />

<!-- 受控模式 -->
<SmartInput v-model="formData.username" />
```

### 注意事项
- 受控模式下，必须确保外部状态更新是同步的，否则可能出现光标跳动问题
- 对于复杂输入（如富文本编辑器），受控模式可能带来性能问题
- 提供清晰的 API 文档，说明两种模式的使用场景
- 可以使用 `useVModel` 等 VueUse 工具简化实现

## 4. 函数式组件 + JSX 的轻量模式

### 核心思想
对于纯展示型组件或需要高性能渲染的场景，使用函数式组件配合 JSX 可以获得更小的运行时开销和更灵活的渲染控制。

### 适用场景
- 高频渲染的列表项组件
- 纯展示型组件（无状态）
- 需要动态渲染逻辑的组件
- 性能敏感场景

### 代码示例

```tsx
import type { PropType } from 'vue'
// FunctionalList.tsx
import { defineComponent } from 'vue'

interface Item {
  id: string | number
  title: string
  description?: string
}

// 使用 defineComponent + 函数式写法
export const FunctionalList = defineComponent(
  (props: { items: Item[], renderItem?: (item: Item) => any }) => {
    return () => (
      <ul class="functional-list">
        {props.items.map(item => (
          <li key={item.id} class="list-item">
            {props.renderItem
              ? (
                  props.renderItem(item)
                )
              : (
                  <>
                    <span class="title">{item.title}</span>
                    {item.description && (
                      <span class="desc">{item.description}</span>
                    )}
                  </>
                )}
          </li>
        ))}
      </ul>
    )
  },
  {
    props: {
      items: { type: Array as PropType<Item[]>, required: true },
      renderItem: Function as PropType<(item: Item) => any>
    }
  }
)

// 完全函数式的写法（更简洁）
export function SimpleTag(props: { text: string, type?: 'primary' | 'danger' }) {
  return <span class={['tag', `tag-${props.type ?? 'default'}`]}>{props.text}</span>
}
```

**在 SFC 中使用：**

```vue
<script setup lang="ts">
import { FunctionalList } from './FunctionalList'

const items = [
  { id: 1, title: 'Vue 3', description: '渐进式框架' },
  { id: 2, title: 'React', description: '声明式 UI 库' }
]
</script>

<template>
  <FunctionalList :items="items">
    <template #renderItem="{ item }">
      <strong>{{ item.title }}</strong>
      <em>{{ item.description }}</em>
    </template>
  </FunctionalList>
</template>
```

### 注意事项
- 函数式组件没有 `this`，无法使用组件实例方法
- 在 Vue 3 中，函数式组件的性能优势已不如 Vue 2 明显，需权衡可读性
- JSX 需要配置编译工具（如 `@vitejs/plugin-vue-jsx`）
- 对于复杂逻辑，仍建议使用 SFC 以获得更好的可维护性

## 5. 组件 API 设计：渐进式复杂度

### 核心思想
组件 API 应该遵循"简单的事情简单做，复杂的事情可能做"的原则，提供分层级的 API 设计：默认行为开箱即用，高级功能按需启用。

### 适用场景
- 通用组件库设计
- 需要同时服务新手和高级用户的组件
- 需要平衡易用性和灵活性的场景

### 代码示例

```vue
<!-- ProgressiveButton.vue -->
<script setup lang="ts">
import { computed, useSlots } from 'vue'

// Level 1: 最简单的使用方式 - 只需要 label
const props = withDefaults(
  defineProps<{
    // 基础
    label?: string
    type?: 'primary' | 'danger' | 'default'

    // Level 2: 常用配置
    loading?: boolean
    disabled?: boolean
    icon?: string

    // Level 3: 高级定制
    renderType?: 'button' | 'link' | 'custom'
    as?: string | object // 渲染为其他元素或组件
  }>(),
  {
    type: 'default',
    renderType: 'button'
  }
)

const emit = defineEmits<{
  click: [event: MouseEvent]
}>()

const slots = useSlots()

// 根据 renderType 决定渲染内容
const componentTag = computed(() => {
  if (props.as) return props.as
  if (props.renderType === 'link') return 'a'
  return 'button'
})
</script>

<template>
  <component
    :is="componentTag"
    class="btn"
    :class="[`btn-${type}`, { 'btn-loading': loading, 'btn-disabled': disabled }]"
    :disabled="disabled || loading"
    @click="emit('click', $event)"
  >
    <!-- 支持多种内容注入方式 -->
    <template v-if="slots.default">
      <slot />
    </template>
    <template v-else-if="slots.icon">
      <slot name="icon" />
      <span>{{ label }}</span>
    </template>
    <template v-else>
      <span v-if="icon" class="icon" :class="icon" />
      <span v-if="loading" class="spinner" />
      <span>{{ label }}</span>
    </template>
  </component>
</template>
```

**使用示例：**

```vue
<!-- Level 1: 最简单 -->
<ProgressiveButton label="提交" />

<!-- Level 2: 常用配置 -->
<ProgressiveButton label="保存" type="primary" :loading="saving" />

<!-- Level 3: 完全定制 -->
<ProgressiveButton as="router-link" to="/home">
  <template #icon><HomeIcon /></template>
  首页
</ProgressiveButton>
```

### 注意事项
- 保持默认行为的合理性，让 80% 的场景无需配置
- 使用 TypeScript 的可选参数和默认值清晰表达 API 层级
- 文档中应按复杂度分层展示用法
- 避免过度设计，不要为"可能的"需求增加 API 复杂度

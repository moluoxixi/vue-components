## 1. Renderless 组件：逻辑复用的经典模式

### 核心思想
Renderless 组件不渲染任何自身的 DOM 元素，仅通过默认插槽将响应式状态和操作方法传递给消费者。与 Headless 组件类似，但更强调"零 UI 渲染"，纯粹作为逻辑容器存在。

### 适用场景
- 跨项目的通用交互逻辑复用（如拖拽、虚拟滚动、无限加载）
- 需要将同一套逻辑应用到完全不同的 UI 上
- 团队希望统一交互行为但允许各业务方自由定制外观

### 代码示例

```vue
<!-- RenderlessHover.vue -->
<script setup lang="ts">
import { ref } from 'vue'

const isHovered = ref(false)

const onMouseEnter = () => { isHovered.value = true }
const onMouseLeave = () => { isHovered.value = false }
</script>

<template>
  <div
    @mouseenter="onMouseEnter"
    @mouseleave="onMouseLeave"
    style="display: contents"
  >
    <slot :is-hovered="isHovered" />
  </div>
</template>
```

**消费者使用：**

```vue
<RenderlessHover v-slot="{ isHovered }">
  <div
    class="card"
    :class="{ 'card--highlight': isHovered }"
    :style="{ transform: isHovered ? 'scale(1.02)' : 'scale(1)' }"
  >
    <h3>悬停试试</h3>
    <p v-if="isHovered">你正在悬停此卡片</p>
  </div>
</RenderlessHover>
```

### 注意事项
- 使用 `display: contents` 让包裹容器不占据布局空间，避免破坏消费者的样式结构
- Vue 3 中 `display: contents` 在某些浏览器下与 CSS Grid/Flex 的交互可能有兼容问题
- 如果逻辑较复杂且不需要模板，优先考虑抽取为 Composable 而非 Renderless 组件
- Renderless 组件适合需要"包裹"的场景（如绑定事件到容器），Composable 更适合纯逻辑复用

## 2. 动态组件 + 异步加载的插件化架构

### 核心思想
利用 Vue 3 的 `<component :is>` 配合 `defineAsyncComponent`，实现组件的运行时动态注册和按需加载，让应用具备"插件化"扩展能力，无需硬编码所有可能的组件。

### 适用场景
- 低代码平台 / 表单设计器的组件动态渲染
- 仪表盘系统，用户可自由配置面板组件
- 多租户 SaaS 中不同客户加载不同功能模块
- 微前端场景下的远程组件加载

### 代码示例

```ts
// registry.ts — 组件注册中心
import { defineAsyncComponent, type Component, shallowRef } from 'vue'

const componentMap = shallowRef<Record<string, Component | string>>({
  chart: defineAsyncComponent(() => import('./ChartPanel.vue')),
  table: defineAsyncComponent(() => import('./TablePanel.vue')),
  // 默认内置组件
})

// 运行时注册新组件
export function registerComponent(name: string, loader: () => Promise<any>) {
  componentMap.value = {
    ...componentMap.value,
    [name]: defineAsyncComponent(loader),
  }
}

// 按名称获取组件
export function getComponent(name: string) {
  return componentMap.value[name]
}
```

```vue
<!-- DynamicRenderer.vue -->
<script setup lang="ts">
import { getComponent } from './registry'

const props = defineProps<{
  componentName: string
  componentProps?: Record<string, any>
}>()
</script>

<template>
  <component
    :is="getComponent(componentName)"
    v-bind="componentProps"
  >
    <slot />
  </component>
</template>
```

```vue
<!-- 业务中使用 -->
<script setup lang="ts">
import { registerComponent } from './registry'

// 运行时动态注册
registerComponent('custom-widget', () => import('./CustomWidget.vue'))
</script>

<template>
  <DynamicRenderer component-name="chart" :component-props="{ data: chartData }" />
  <DynamicRenderer component-name="custom-widget" />
</template>
```

### 注意事项
- `shallowRef` 用于存储组件映射，避免 Vue 对大量组件对象进行深度代理带来的性能开销
- 异步组件加载失败时，通过 `defineAsyncComponent` 的 `errorComponent` 和 `timeout` 提供降级体验
- 动态组件的 TypeScript 类型推导较弱，可通过泛型或类型断言增强
- 注意组件卸载时的内存清理，特别是远程加载的组件

## 3. 事件总线 Composable：解耦跨组件通信

### 核心思想
基于 Vue 3 的响应式系统实现一个类型安全的事件总线，封装为 Composable 供全局使用。相比 Vue 2 的 `$on/$off`，这种方案更符合 Composition API 的风格，且天然支持 TypeScript 类型推导。

### 适用场景
- 兄弟组件或跨层级组件之间的松耦合通信
- 插件/模块系统中的消息通知
- 需要解耦但又不值得引入 Pinia 的轻量状态同步

### 代码示例

```ts
// useEventBus.ts
import { reactive, type UnwrapRef } from 'vue'

type EventMap = Record<string, any[]>

// 全局事件存储（模块单例）
const bus = reactive<Record<string, Set<(...args: any[]) => void>>>({})

export function useEventBus<T extends EventMap>() {
  function on<K extends keyof T & string>(
    event: K,
    handler: (...args: T[K]) => void
  ) {
    if (!bus[event]) bus[event] = new Set()
    bus[event].add(handler)
    return () => off(event, handler)
  }

  function off<K extends keyof T & string>(
    event: K,
    handler: (...args: T[K]) => void
  ) {
    bus[event]?.delete(handler)
  }

  function emit<K extends keyof T & string>(
    event: K,
    ...args: T[K]
  ) {
    bus[event]?.forEach(handler => handler(...args))
  }

  function once<K extends keyof T & string>(
    event: K,
    handler: (...args: T[K]) => void
  ) {
    const wrapper = (...args: T[K]) => {
      handler(...args)
      off(event, wrapper)
    }
    on(event, wrapper)
  }

  return { on, off, emit, once }
}
```

**定义业务事件类型并使用：**

```ts
// 定义事件映射（全局共享）
interface AppEvents {
  'notification:show': [message: string, type: 'success' | 'error']
  'user:logout': []
  'theme:change': [theme: 'light' | 'dark']
}

// 在任何组件中使用
const { on, emit } = useEventBus<AppEvents>()

// 监听
on('theme:change', (theme) => {
  console.log('主题切换为:', theme)
})

// 触发
emit('notification:show', '保存成功', 'success')
```

### 注意事项
- `on` 返回取消订阅函数，建议在 `onUnmounted` 中调用以避免内存泄漏
- 事件名建议使用命名空间格式（如 `module:action`），避免冲突
- 对于复杂的状态共享场景，仍应优先使用 Pinia 等状态管理方案
- 调试困难是事件总线的通病，建议在开发环境添加事件日志中间件

## 4. 插槽代理模式：优雅的组件二次封装

### 核心思想
在二次封装第三方组件时，通过 `$slots` 和 `$attrs` 的透传机制，让封装组件保持与原组件一致的插槽和属性接口，消费者无需感知封装层的存在。

### 适用场景
- 对第三方 UI 库组件进行业务定制（如统一加埋点、加权限控制）
- 团队内部基于基础组件扩展通用行为
- 需要在不修改原组件源码的情况下增强功能

### 代码示例

```vue
<!-- EnhancedDialog.vue — 对 Element Plus ElDialog 的二次封装 -->
<script setup lang="ts">
import { useSlots, useAttrs, computed } from 'vue'
import { ElDialog } from 'element-plus'

const props = defineProps<{
  trackName?: string  // 增加埋点标识
  permission?: string  // 增加权限控制
}>()

const slots = useSlots()
const attrs = useAttrs()

// 权限判断
const hasPermission = computed(() => {
  if (!props.permission) return true
  return checkPermission(props.permission)
})

// 埋点上报
function handleOpened() {
  if (props.trackName) {
    track('dialog_open', { name: props.trackName })
  }
  attrs.onOpened?.()
}
</script>

<template>
  <ElDialog
    v-bind="attrs"
    :model-value="hasPermission ? attrs.modelValue : false"
    @opened="handleOpened"
  >
    <!-- 透传所有插槽 -->
    <template v-for="(_, name) in slots" :key="name" #[name]="slotProps">
      <slot :name="name" v-bind="slotProps ?? {}" />
    </template>
  </ElDialog>
</template>
```

**消费者无感知使用：**

```vue
<EnhancedDialog
  v-model="visible"
  track-name="edit-profile"
  permission="profile:edit"
  title="编辑资料"
>
  <template #default>
    <EditForm />
  </template>
  <template #footer>
    <button @click="visible = false">取消</button>
    <button @click="save">保存</button>
  </template>
</EnhancedDialog>
```

### 注意事项
- `v-for` 遍历 `$slots` 时会包含 Vue 内部插槽（如 `__` 前缀的），可通过过滤排除
- `v-bind="attrs"` 会透传所有未声明的属性和事件，注意避免与封装层自身的属性冲突
- 使用 `inheritAttrs: false` 防止 attrs 被自动绑定到根元素
- 对第三方组件的版本升级可能导致插槽接口变化，建议编写集成测试覆盖关键插槽

## 5. 虚拟化列表组件的抽象设计

### 核心思想
将虚拟滚动的核心逻辑（可视区域计算、滚动位置追踪、数据切片）抽象为 Composable，再通过组件包装提供声明式 API，实现逻辑与 UI 的分离，同时支持不同的渲染策略。

### 适用场景
- 大数据量列表/表格渲染（1000+ 条数据）
- 需要支持多种滚动容器（window、自定义容器、嵌套滚动）
- 虚拟滚动 + 分页加载的组合场景

### 代码示例

```ts
// useVirtualList.ts
import { ref, computed, onMounted, onUnmounted, type Ref } from 'vue'

interface UseVirtualListOptions {
  container: Ref<HTMLElement | null>
  itemHeight: number | ((index: number) => number)
  overscan?: number
}

export function useVirtualList<T>(
  items: Ref<T[]>,
  options: UseVirtualListOptions
) {
  const { container, itemHeight, overscan = 5 } = options

  const scrollTop = ref(0)
  const containerHeight = ref(0)

  // 获取指定索引项的高度
  const getHeight = (index: number) =>
    typeof itemHeight === 'function' ? itemHeight(index) : itemHeight

  // 计算总高度
  const totalHeight = computed(() =>
    items.value.reduce((sum, _, i) => sum + getHeight(i), 0)
  )

  // 计算偏移量数组（每项的 top 位置）
  const offsets = computed(() => {
    const arr: number[] = []
    let offset = 0
    for (let i = 0; i < items.value.length; i++) {
      arr.push(offset)
      offset += getHeight(i)
    }
    return arr
  })

  // 二分查找：根据 scrollTop 找到起始索引
  const startIndex = computed(() => {
    let low = 0
    let high = items.value.length - 1
    while (low <= high) {
      const mid = Math.floor((low + high) / 2)
      if (offsets.value[mid] <= scrollTop.value) {
        low = mid + 1
      } else {
        high = mid - 1
      }
    }
    return Math.max(0, low - 1)
  })

  const endIndex = computed(() => {
    let accumulated = 0
    let idx = startIndex.value
    while (idx < items.value.length && accumulated < containerHeight.value) {
      accumulated += getHeight(idx)
      idx++
    }
    return Math.min(items.value.length, idx + overscan)
  })

  const visibleItems = computed(() =>
    items.value.slice(startIndex.value, endIndex.value)
  )

  const offsetY = computed(() => offsets.value[startIndex.value] ?? 0)

  // 滚动监听
  function onScroll() {
    if (container.value) {
      scrollTop.value = container.value.scrollTop
    }
  }

  onMounted(() => {
    container.value?.addEventListener('scroll', onScroll, { passive: true })
    if (container.value) {
      containerHeight.value = container.value.clientHeight
    }
  })

  onUnmounted(() => {
    container.value?.removeEventListener('scroll', onScroll)
  })

  return {
    totalHeight,
    visibleItems,
    offsetY,
    startIndex,
    endIndex,
  }
}
```

```vue
<!-- VirtualList.vue -->
<script setup lang="ts" generic="T">
import { ref } from 'vue'
import { useVirtualList } from './useVirtualList'

const props = withDefaults(
  defineProps<{
    items: T[]
    itemHeight: number | ((index: number) => number)
    overscan?: number
  }>(),
  { overscan: 5 }
)

const containerRef = ref<HTMLElement | null>(null)

const { totalHeight, visibleItems, offsetY } = useVirtualList(
  () => props.items,
  { container: containerRef, itemHeight: props.itemHeight, overscan: props.overscan }
)
</script>

<template>
  <div ref="containerRef" class="virtual-list-container" style="overflow-y: auto; height: 100%">
    <div class="virtual-list-phantom" :style="{ height: `${totalHeight}px` }">
      <div class="virtual-list-content" :style="{ transform: `translateY(${offsetY}px)` }">
        <slot
          v-for="(item, index) in visibleItems"
          :key="index"
          :item="item"
          :index="index"
        />
      </div>
    </div>
  </div>
</template>
```

### 注意事项
- 使用 `passive: true` 的滚动监听避免阻塞主线程
- 对于动态高度的列表项，需要实现高度缓存机制避免反复计算
- 滚动容器的 `height` 必须有明确值，否则无法正确计算可视区域
- 搜索/过滤场景下，切换数据后应重置 `scrollTop` 到顶部
- 生产环境建议使用成熟方案（如 `@tanstack/vue-virtual`），此示例主要用于理解原理

---

> 💡 **今日思考**：组件设计的本质是"职责分离"。无论是 Renderless 组件、Headless 组件还是 Composable，核心思路都是将"做什么"与"怎么展示"解耦。选择哪种模式，取决于你的逻辑是否需要 DOM 包裹——如果需要（如绑定事件、监听滚动），用 Renderless；如果不需要，用 Composable 更纯粹。
